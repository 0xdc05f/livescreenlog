# Open-source publish checklist

Use this before the first public `git init` / push, and before every release that might contain new secrets or local tooling.

## Must not publish

| Path / pattern | Why |
|----------------|-----|
| `.env`, `.env.*` (except `.env.example`) | Real secrets |
| `deploy/.env`, `deploy/docker-compose.override.yml` | Local operator overrides |
| `*.pem`, `*.key`, `*.jks`, `*.p12`, keystores | Certificates / private keys |
| `credentials.json`, cloud tokens, `FIGMA_API_KEY`, `SNYK_TOKEN` | Third-party credentials |
| Production HMAC / project API keys | Auth bypass if leaked |
| Real user session dumps, DB dumps (`*.dump`, `*.sql.gz`, `*.rdb`) | PII / customer data |
| `.kilo/`, `.ai/`, `.cursor/`, `.claude/`, agent session IDs | Local AI state |
| `.vscode/`, `.idea/` | Machine-specific IDE settings |
| `build/`, `bin/`, `.gradle/`, `*.class`, `*.jar` (except wrapper) | Build outputs |
| `node_modules/`, `sdk/dist/`, `frontend/dist/` | Dependencies / build artifacts |
| `src/main/resources/static/{assets,index.html,livescreenlog.js,sample*.html,...}` | Generated by Gradle FE/SDK tasks |
| `.github/modernize/` | Internal modernization hooks |
| `tasks/` | Internal phase task notes (not user docs) |

These are covered by the root `.gitignore`. Verify with a dry-run before the first commit:

```bash
git status --ignored
# or, after git init:
git add -n .
```

## Safe to publish

| Path | Notes |
|------|--------|
| Source under `src/`, `frontend/src/`, `sdk/src/` | Core product |
| `build.gradle`, `settings.gradle`, `gradlew*`, `gradle/wrapper/` | Build system |
| `deploy/Dockerfile`, `deploy/docker-compose.yml` | **Reference only** — operators customize |
| `.env.example` | Placeholders only |
| `docs/**` | Public documentation (includes `docs/deploy/DEPLOY.md`) |
| `sample*.html` at **repo root** | Demo pages (not the copied static copies) |
| `LICENSE`, `README.md`, `SECURITY.md`, `CONTRIBUTING.md` | Community files |
| `kilo.json` | Optional local MCP config; **no secrets** (keys via env only) |
| `version.json`, `release.sh` | Versioning helpers |

## Secrets audit (run locally)

```bash
# Real env files
find . -name '.env' -o -name '.env.local' 2>/dev/null

# Common secret shapes (review hits carefully; false positives are normal)
grep -RInE --exclude-dir=node_modules --exclude-dir=.gradle --exclude-dir=build \
  --exclude-dir=bin --exclude-dir=.git \
  '(api[_-]?key|secret|password|token|BEGIN [A-Z ]*PRIVATE|sk-|ghp_|github_pat|AKIA)' .
```

Confirm:

1. No real `.env` in the tree.
2. `application-*.yml` only uses env placeholders or **local-dev** defaults (not production passwords).
3. `deploy/docker-compose.yml` defaults are for local/reference only; production must override via env.
4. Sample HTML uses demo `projectKey` values, not live keys.
5. No personal email, private company URLs, or customer identifiers in docs/samples.

## Default credentials (local only)

These appear intentionally for local Docker/dev. **Change them before any shared or public deployment.**

| Location | Values |
|----------|--------|
| `application-dev.yml` / compose Postgres | `livescreenlog_user` / `livescreenlog_password` |
| `application.yml` defaults | weak HMAC / dashboard password placeholders |
| `deploy/docker-compose.yml` app service | compose-local HMAC default |

Production (`prod` profile) should fail fast on weak HMAC and wildcard capture origins. See `docs/security/SECURITY.md`.

## First public commit suggested flow

```bash
# 1. Clean generated / local junk (optional if already gitignored)
./gradlew clean
rm -rf frontend/node_modules sdk/node_modules sdk/dist build bin .gradle

# 2. Init and review
git init
git add -n .          # dry-run: nothing sensitive should appear
git status

# 3. Commit public surface only
git add .
git status            # final review
git commit -m "Initial public release"

# 4. Add remote and push (after creating empty GitHub repo)
# git remote add origin git@github.com:<org>/<repo>.git
# git push -u origin main
```

## After publish

- [ ] Enable private vulnerability reporting on GitHub.
- [ ] Add branch protection / required checks if CI is added.
- [ ] Rotate any keys that ever lived on this machine in a real `.env` (even if never committed).
- [ ] Do not force-push history that once contained secrets; if secrets were committed, rotate and consider history rewrite **before** wide distribution.
